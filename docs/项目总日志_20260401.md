## 一、项目目标
- 小车在已知二维地图中按固定航点巡航。
- 使用单目相机和视觉链识别闯入的陌生人。
- 检测到 `unknown` 后中断巡航并转入跟随。
- 目标离开或丢失后，基于当前地图位姿回到最近巡逻点，继续原巡航任务。
- 整套系统尽量复用现有 ROS2 / Nav2 / 底盘控制链，而不是推倒重做。

## 五、当前已确认的关键结论

### 1. 控制权切换是核心工程点
- Nav2 / `car_controller_new.cpp` 会发 `/car_cmd_vel`
- `face_follow_controller.py` 也会发 `/car_cmd_vel`
- 所以比赛版必须有一个明确的“主管层”来裁决控制权。
- 当前 V1 方案复用了现有 `/light` 接口：
  - `red`：压住导航控制并中位停车
  - `green`：恢复导航控制

### 2. 追踪后能回巡航的根本原因
- 跟随时暂停的是“导航控制权”，不是“定位系统”。
- 激光、TF、AMCL 在跟随期间仍持续估计 `map -> base_link`。
- 因此目标丢失后，小车能按当前地图位姿回最近航点，而不是盲回。

### 3. 当前最稳的工程方向
- 视觉负责“看见谁、认出谁、当前跟谁”。
- 导航负责“怎么安全走、怎么绕障碍、怎么回路线”。
- 安全层尽量由雷达主导，而不是依赖视觉。
- 后续成熟形态应该逐步朝“动态目标导航”演进，而不是长期视觉直接控底盘。

## 九、基于学习后的系统级理解
- 这个项目不是“YOLO 看见人就追”的小脚本拼接。
- 它应被理解成一套基于预建地图的移动安防巡逻机器人系统。

### 推荐系统分层
1. 定位层
   - 激光雷达、IMU、编码器、AMCL、里程计、TF
2. 导航层
   - Nav2、全局规划、局部控制、costmap
3. 视觉层
   - YOLOv5、YuNet、SFace、目标检测与识别
4. 目标管理层
   - 目标筛选、身份判断、轨迹保持、丢失恢复
5. 调度层
   - `PATROL -> ALERT_TRACK -> RETURN_TO_ROUTE -> SAFE_STOP`
6. 安全层
   - 雷达主导的最终速度保护
7. 运维层
   - 预检、录包、诊断、统计、受控启动
